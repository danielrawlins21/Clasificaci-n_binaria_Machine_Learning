\section{Conclusiones}

Esta sección sintetiza hallazgos, limitaciones y recomendaciones finales del trabajo.

\subsection{Síntesis de hallazgos}

El trabajo implementó un pipeline de Machine Learning en tres fases:

\begin{itemize}
  \item \textbf{Fase 1 (auditoría)}: depuración de calidad, deduplicación y control de no-leakage.
  \item \textbf{Fase 2 (selección)}: RFE y validación de candidatas; se consolidó un conjunto compacto de variables.
  \item \textbf{Fase 3 (modelado)}: comparación de RF\_Accuracy (0.949 de accuracy) frente a ET\_Recall (0.943 de recall).
\end{itemize}

\subsection{Limitaciones del estudio}

\begin{itemize}
  \item Tamaño muestral moderado tras depuración ($n=3538$).
  \item Comparación acotada a RF y ET (sin benchmarking amplio de otros algoritmos).
  \item Variables anonimizadas y ausencia de costes de negocio explícitos para optimización económica.
  \item Riesgo de deriva temporal no evaluado en esta entrega.
\end{itemize}

\subsection{Conclusiones metodológicas y prácticas}

\begin{enumerate}
  \item La auditoría inicial y el control de leakage son determinantes para la validez de resultados.
  \item La selección de variables dentro de CV y la búsqueda de threshold mejoran robustez e interpretabilidad.
  \item La comparación multiobjetivo (accuracy vs recall) evita decisiones sesgadas por una única métrica.
  \item Para despliegue, \textbf{RF\_Accuracy} es el modelo principal; \textbf{ET\_Recall} queda como apoyo de cobertura.
  \item Se recomienda monitorizar periódicamente accuracy/recall y recalibrar threshold cuando cambie el contexto.
\end{enumerate}

\subsection{Trabajo futuro}

\begin{enumerate}
  \item Ampliar el benchmarking con boosting (XGBoost/LightGBM) y SVM.
  \item Incorporar técnicas de desbalance y calibración probabilística.
  \item Añadir explicabilidad (SHAP/LIME) y validación de impacto de negocio (A/B test).
\end{enumerate}

\subsection{Cierre}

El churn resulta predecible con buen rendimiento cuando se combina depuración rigurosa, selección de variables, validación estratificada y ajuste explícito de threshold. El flujo propuesto queda listo para uso operativo supervisado y mejora iterativa.
